▶ How to add MT5 accounts (multi-account support)  — click to expand
Step 1 — Add account on MetaAPI
1. Go to app.metaapi.cloud → Accounts → Add Account
2. Platform: MT5
3. Broker server: Tickmill-Demo (or Tickmill-Live)
4. Login: your account number (e.g. 25329025)
5. Password: your MT5 password
6. Click Deploy — copy the Account ID it generates
Step 2 — Set Railway env vars
Set METAAPI_ACCOUNTS as a JSON array on Railway:
[
  {"id":"<metaapi-id-1>","label":"Demo 1 (#25329025)"},
  {"id":"<metaapi-id-2>","label":"Demo 2 (#99999)"}
]
Keep METAAPI_TOKEN — it covers all accounts under the same MetaAPI user.
METAAPI_ACCOUNT_ID still works for a single account (backward compat).
Step 3 — Select account & start
• After setting env vars, redeploy on Railway and refresh this page
• Use the account dropdown (top-right of MT5 panel) to pick which account to trade on
• Each account trades independently — open multiple browser tabs to monitor both simultaneously
• Risk sizing uses that account's live MT5 equity — each account sizes its own trades

Strategy Logic — NY Session 4H Level + 5M Flag/Reclaim · MetaAPI MT5 Execution
Step 1 · Find NY Session 4h Level
• Find the first 4h candle of the New York session each day
• NY session 4h candle = opens at 12:00 UTC, closes at 16:00 UTC
• LevelHigh = High · LevelLow = Low of that candle
Step 2 · Find FLAG Candle (after 16:00 UTC)
• After the 4h candle closes, switch to the 5-minute chart
• Watch for a 5m candle whose close < LevelLow
• That candle becomes the FLAG candle
Step 3 · BUY Entry (LevelLow Reclaim)
• After FLAG candle forms, watch for 5m close > LevelLow
• SL = Lowest low of all 5m candles from FLAG → entry
• Size = (MT5 Equity × 2%) / SL-dist | capped 0.01–10 lots
• Risk per trade: 2% of equity · Leverage: 1:500
Step 4 · Decremental SL Trail (5s checks)
• 1R profit → SL = entry (breakeven)
• 1.5R profit → SL = entry + 1×dist (1R locked)
• 2R profit → Phase 4 trail: SL = max(price) − min($25, 0.25×dist)
• SL changes sync to MT5 via POSITION_MODIFY in live mode
